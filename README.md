# FUTURE_CS_03

## API Security Risk Analysis

This repository contains Task 3 submission for the Cyber Security Internship at Future Interns.

## Project Objective

The objective of this task is to understand API communication, test API responses using Postman, identify common API security risks, and improve cybersecurity awareness.

## Tools Used

* Postman
* Browser
* Microsoft Word

## APIs Tested

1. https://jsonplaceholder.typicode.com/users
2. https://jsonplaceholder.typicode.com/posts

## Activities Performed

* API request testing using GET requests
* Response analysis using Postman
* Status code verification
* Identification of API security risks
* Documentation and reporting

## Risks Identified

* No Authentication
* Public Data Exposure
* Open API Access
* No Rate Limiting

## Learning Outcome

Through this task, I learned how APIs communicate using HTTP requests and responses, how to test APIs using Postman, and how to identify common API security risks.

## Conclusion

The tested APIs responded successfully and returned JSON-formatted data using GET requests. Several API security concerns were identified, including lack of authentication and unrestricted public access.
